# 📘 Introduction to Project Management for Software Development

This course connects **how systems are built** with **how projects are managed**.

Students will learn:

* How IT systems work (infrastructure → cloud)
* How software is built (SDLC)
* How projects are managed (Waterfall vs Agile)

---

# Day - 1

# 🌐 Understanding IT Infrastructure & Systems

## What is IT Infrastructure?

IT infrastructure is the foundation of all software systems.

Includes:

* Routers, Switches
* Firewalls, VPN
* Web Servers, DB Servers
* FTP Servers, Backup Systems

---

## Key Concepts Covered

* Internal vs External Access (VPN)
* B2B Communication (FTP, APIs)
* Network Security (Whitelisting, Firewalls)
* Data Flow (Packets, Protocols)

---

## Real-World Example

* Employees connect internally
* Vendors connect via VPN
* Businesses exchange data securely

# Day - 2

# 🏗️ System Architecture & Modern Computing

## 1. Distributed Systems

* Data across multiple locations

* Replication (Push/Pull)

![replication](/Assets/Arch-2.pdf)

### Pros

* High availability
* Scalable

### Cons

* Complexity
* Data inconsistency

---

## 2. Centralized Systems

* Single web app + database

### Pros

* Simpler design
* Real-time access

### Cons

* Single point of failure
* Scaling limits

---

## 3. Virtual Machines (VMs)

* Multiple systems on one machine

### Pros

* Faster provisioning
* Isolation

### Cons

* Maintenance required

---

## 4. Containers

* Lightweight environments (Docker)

### Pros

* Fast deployment
* Scalable

### Cons

* Complexity

---

## 5. Cloud Computing

### What is Cloud?

Cloud = Infrastructure delivered over the internet (pay-as-you-go)

### Key Idea

**Cloud is still physical infrastructure — just managed and scalable**

# Day - 3

# 🔄 Software Development Lifecycle (SDLC)

## What is SDLC?

The process used to build software systems.

---

## SDLC Phases

1. Requirements
2. Design
3. Development
4. Testing
5. Deployment
6. Maintenance

---

## Real Example

Building a secure vendor platform:

* Requirements → What problem?
* Design → Central vs Distributed?
* Development → Build system
* Testing → Security + performance
* Deployment → Release
* Maintenance → Updates

---

## SDLC + Architecture Connection

Different systems require different planning approaches.

# Day - 4

# ⚙️ Project Management Methodologies

## Waterfall Methodology

### Characteristics

* Linear process
* Fixed requirements
* Heavy planning

### Best For

* Infrastructure projects
* Stable environments

---

## Agile Methodology

### Characteristics

* Iterative development
* Continuous feedback
* Flexible requirements

### Best For

* Cloud systems
* Modern applications

---

## Key Comparison

| Feature     | Waterfall | Agile      |
| ----------- | --------- | ---------- |
| Planning    | Upfront   | Continuous |
| Flexibility | Low       | High       |
| Delivery    | One-time  | Frequent   |

---

## Key Insight

**The type of system determines the project approach**
